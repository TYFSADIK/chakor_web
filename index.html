Three things you will not find stitched together anywhere else.

Most local AI tools make you find out the hard way. Chakor tells you up front, then gets out of your way.

01

It knows what your machine can run

Chakor reads your RAM and GPU first, then tags every model so you never load one that crashes. A 4 GB laptop and a 24 GB workstation each get an honest answer.

fits tight too big
02

Get models without touching a terminal

Search Hugging Face inside the app, see every quant with its size and a fit tag, and click once. It downloads in the background, straight into your models folder. Close the tab and it keeps going.

downloading qwen2.5-14b · 64% · 28 MB/s
03

Switch engines without the crashes

Flip between llama.cpp, Ollama, and LM Studio in one tap. Chakor unloads the previous model first, so two models never fight for the same VRAM. Roomy GPU? Keep several loaded.

llama.cpp → ollama · freed 9.0 GB · loaded
everything it does

One workspace. A lot of capabilities.

Started as an AI chat. Became a workspace. Each piece runs locally against whatever endpoints you point it at.

models

Local or cloud, your call

Run local through Ollama, LM Studio, or llama.cpp on GPU or plain CPU. Or paste keys for OpenAI, Anthropic, Google, or OpenRouter.

models

Switch per message

Pick any model from the chat header and the next message uses it. Swap the local model or its context size from the UI, no restart.

hardware

Fit tags for every model

Chakor reads your RAM and GPU and marks each model FITS, TIGHT, or TOO BIG, with a recommended pick. No more crash-to-find-out.

web

Search built in

Turn it on per message. SearXNG first, then Brave, then DuckDuckGo as fallback, so a single dead key never stops you.

documents

Talk to your files

Drop in PDFs, text, or markdown and ask questions grounded in them. Answers cite what they came from.

research

Real research mode

Plans angles, gathers sources, reads them, and writes a cited briefing instead of a single guessy answer.

compare

Blind A/B, with a leaderboard

Send one prompt to several models at once, vote on the answers blind, and keep a running leaderboard.

memory

It remembers what matters

The assistant keeps facts you want it to and recalls them across conversations. You decide what sticks.

notes

A notepad that lives in the app

Keep-style notes with checklists, colors, pinning, and archive, right next to your chats.

vision

See your images

On models that support it, attach a picture and ask about it. The attach button appears on its own.

private

No telemetry, ever

Conversations stay in a local SQLite database on your disk. Nothing phones home unless you turn on search or a cloud model.

private

Multi-user and yours to rebrand

Accounts with an admin panel, a live theme editor, and a config file that renames the whole app. Installs as a phone app too.

why this exists

Privacy is not a setting you toggle.

Every big AI app wants the same deal. You hand over your questions, your documents, your half formed ideas at 2am, and they keep all of it on servers you will never see.

Self hosting flips that. You run the model. You hold the data. No account to harvest, no telemetry phoning home, no data broker in the middle, and no government letter they can answer with your chat history, because they never had it.

It is where the software runs. Chakor runs on your machine, talks to nobody you did not ask it to, and still tries to be genuinely nice to use. That is the whole pitch.

you@yourmachine: ~/chakor
get started

Pick whichever fits you.

All three end with the app on http://localhost:3001. The first account you register becomes the admin.

The least fuss. Install Docker, then run two lines. A secret key is generated for you. Want a model bundled in too? docker compose --profile ollama up -d.

$ git clone https://github.com/TYFSADIK/chakor.git && cd chakor
$ docker compose up -d

The hands-off path. The installer works out your OS and package manager, installs Git, a build chain, Node 20+, and Ollama, then starts Chakor. Add --all for a local llama.cpp build with CUDA auto-detected. On Android, run it inside Termux and your phone becomes the server.

$ git clone https://github.com/TYFSADIK/chakor.git && cd chakor
$ bash install.sh # or: bash install.sh --all

For developers. Set AUTH_SECRET in .env.local with openssl rand -base64 32, then build and start.

$ git clone https://github.com/TYFSADIK/chakor.git && cd chakor
$ npm install --legacy-peer-deps
$ cp .env.example .env.local # then set AUTH_SECRET
$ npm run build && npm start
